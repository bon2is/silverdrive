# 개선 명세서 — 실제 적성검사 항목 적용 + AdSense 실구현

## 1. 실제 고령 운전자 인지능력 검사 분석

**근거:** 도로교통공단(KOROAD) 고령 운전자 적성검사 항목 (2024 기준)

| 실제 검사 항목 | 측정 능력 | PWA 구현 방식 |
|--------------|---------|------------|
| 단어 기억 회상 | 기억력 | 6개 단어 → 회상 선택 |
| 숫자 순서 연결 (Trail Making A) | 주의력·집중력 | 1~10 숫자 순서대로 탭 |
| 신호 반응 검사 | 반응속도·억제 | 초록불만 탭, 빨간불 무시 |
| 표지판 식별 | 시각 인식 | 기존 + 실제 표지판 확대 |
| 위험 지각 | 판단력 | 기존 유지 |

## 2. 새 테스트 전체 흐름

```
/test (허브)
 └─ 전체 시작
      ├─ /test/memory   ① 기억력 검사    (기억 10초 → 회상)
      ├─ /test/trail    ② 숫자 연결 검사 (1→10 순서 탭)
      ├─ /test/reaction ③ 신호 반응 검사 (초록불만 탭, 5회)
      ├─ /test/signs    ④ 표지판 식별    (5문항)
      └─ /test/hazard   ⑤ 위험 지각      (3회)
           └─ /result-loading → /result
```

## 3. 기억력 검사 상세 명세 (`/test/memory`)

**Phase A — 기억하기 (10초)**
- 6개 단어를 2×3 그리드로 동시 표시
- 10초 카운트다운 표시
- TTS: "화면의 단어 6개를 잘 기억해주세요"

**단어 풀 (20개, 라운드마다 6개 랜덤 선택):**
```
과일: 사과, 바나나, 포도, 딸기, 귤
교통: 자동차, 버스, 기차, 자전거, 배
장소: 병원, 학교, 시장, 은행, 약국
사물: 안경, 우산, 열쇠, 지갑, 신발
```

**Phase B — 회상하기**
- 12개 단어 3×4 그리드 (정답 6 + 오답 6)
- "아까 본 단어를 모두 선택해주세요"
- 선택 후 "확인" 버튼
- TTS: "아까 본 단어를 모두 찾아 눌러주세요"

**저장:** `memoryScore: number` (0~6, 맞춘 개수)

## 4. 숫자 연결 검사 상세 명세 (`/test/trail`)

- 숫자 1~10을 화면에 랜덤 위치로 배치 (겹침 없음)
- 각 숫자: 노란 원, 70px, 안에 숫자 표시
- 현재 탭해야 할 숫자는 테두리 강조 표시
- 순서 틀리면: 빨간 플래시 + "X번을 먼저 눌러주세요"
- 완료 시 총 소요 시간(ms) 기록

**저장:** `trailTime: number` (ms), `trailErrors: number`

---

## 5. 신호 반응 검사 개선 (`/test/reaction`)

**기존:** 노란 원 탭
**개선:** 신호등 방식

- 빨간불 표시 (1.5~3초)
- 초록불로 전환 → 즉시 탭
- **빨간불에 탭 = 실수 기록** (억제 능력 측정)
- 초록불 3초 이내 탭 = 성공, 초과 = 실패
- 5라운드

**저장:** `reactionTimes: number[]`, `reactionMistakes: number`

## 7. AdSense 실구현

**환경변수:**
```env
NEXT_PUBLIC_ADSENSE_CLIENT=ca-pub-XXXXXXXXXXXXXXXX   # Publisher ID
NEXT_PUBLIC_ADSENSE_SLOT=XXXXXXXXXX                  # Ad Slot ID (result-loading용)
```

**구현 방식:**
- Publisher ID + Slot ID 모두 있을 때 → 수동 광고 단위 (`<ins>`)
- Publisher ID만 있을 때 → Auto Ads (스크립트만으로 자동 배치)
- 둘 다 없을 때 → 개발 플레이스홀더

**적용 위치:**
- `/result-loading`: 메인 광고 (300×250)
- `/result`: 하단 배너 (향후 Phase 4)
