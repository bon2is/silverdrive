# Phase 2 기능 명세서 — 테스트 엔진

> 작성자: 기획/설계 에이전트 | 2026-03-17

---

## 화면 흐름

```
/ (홈)
 └─ "테스트 시작하기" 탭
      └─ /test (테스트 허브)
           └─ "전체 테스트 시작" 탭
                ├─ /test/reaction  (자극 반응 — 5라운드)
                ├─ /test/signs     (표지판 식별 — 5라운드)
                └─ /test/hazard    (위험 지각 — 3라운드)
                     └─ /result-loading (Phase 3 스텁)
```

---

## 공통 UX 원칙 (전 테스트 적용)

| 원칙 | 구현 방법 |
|------|----------|
| TTS 음성 | 각 화면 진입 시 자동 재생 + 재생 버튼 제공 |
| 큰 글씨 | 지시문 24px, 보기 레이블 20px |
| 터치 타겟 | 모든 탭 영역 최소 80×80px (버튼), 120×120px (테스트 대상) |
| 더블탭 방지 | 한 번 탭 후 disabled 처리 (다음 라운드 준비 중 잠금) |
| 진행 표시 | 상단 고정 바: "3 / 5 완료" 텍스트 + 채워지는 바 |
| 오류 없음 | 시간 초과 = 자동으로 다음 라운드 이동 (실패 표시 후 1.5초) |

## 1. 테스트 허브 `/test`

**목적:** 전체 테스트 흐름 진입점, 각 테스트 간략 안내

**구성:**
- 제목: "운전 적성 자가진단 테스트"
- 3개 테스트 목록 카드 (아이콘 + 이름 + 소요 시간)
- "전체 테스트 시작하기" 버튼 → `/test/reaction` 이동
- TTS: "총 3가지 테스트를 순서대로 진행합니다. 준비되셨으면 시작 버튼을 눌러주세요."

---

## 2. 자극 반응 테스트 `/test/reaction`

**목적:** 반응 속도(ms) 측정

**라운드 수:** 5회

**라운드 흐름:**
1. "준비 화면" — "곧 노란 원이 나타납니다. 보이면 바로 눌러주세요!" (TTS + 텍스트)
2. 랜덤 딜레이 1500~3000ms 대기 (회색 화면)
3. 노란 원(⬤) 출현 — 화면 중앙 ±30% 랜덤 위치, 크기 120px
4. 탭 시 → 반응 시간(ms) 기록, 성공 피드백 (✅ "잘 하셨어요!")
5. 3000ms 초과 시 → 3000ms 기록, 실패 피드백 (⏰ "조금 더 빠르게!")
6. 1500ms 후 다음 라운드

**저장 데이터:** `reactionTimes: number[]` (ms 단위, 5개)

**완료 후:** `/test/signs` 이동

---

## 3. 표지판 식별 테스트 `/test/signs`

**목적:** 표지판 인식력 측정

**라운드 수:** 5회

**표지판 데이터 (7종):**

| id | 이름 | 설명 |
|----|------|------|
| `no-entry` | 진입금지 | 빨간 원 + 흰 가로바 |
| `one-way` | 일방통행 | 파란 사각 + 흰 화살표 → |
| `crosswalk` | 횡단보도 | 삼각형 + 보행자 아이콘 |
| `traffic-light` | 신호등 | 3색 세로 신호등 |
| `speed-limit` | 속도제한 | 빨간 원 + "50" |
| `no-parking` | 주차금지 | 파란 원 + P + 빨간 대각선 |
| `yield` | 양보 | 역삼각형 빨간/흰 |

**라운드 흐름:**
1. 상단: 정답 표지판 이름 텍스트 (24px, bold) + TTS 안내
   - 예: "진입금지 표지판을 찾아 눌러주세요"
2. 2×2 그리드: 4개 보기 (정답 1 + 오답 3, 랜덤 섞음)
3. 탭 시 → 정오답 판별, 피드백 (✅ 정답 / ❌ 오답), 1500ms 후 다음
4. 오답 선택 시에도 정답 강조 표시

**저장 데이터:** `signAnswers: boolean[]` (5개)

**완료 후:** `/test/hazard` 이동

---

## 4. 위험 지각 테스트 `/test/hazard`

**목적:** 위험 인지 속도 측정

**라운드 수:** 3회

**화면 구성 (CSS 전용, 이미지 없음):**
- 회색 배경 도로 씬 (가운데 흰 점선, 양쪽 인도)
- 3초 후 위험 요소(🚶 아이콘) 화면 내 랜덤 위치 출현
- 출현 영역: 전체 화면의 30~70% 범위 (극단 배치 방지)
- 탭 영역: 아이콘 중심 ±60px (충분한 터치 오차 허용)

**라운드 흐름:**
1. "준비 화면" — "도로에 위험 요소가 나타나면 바로 눌러주세요!" (TTS + 텍스트)
2. 씬 표시 → 1000~2000ms 딜레이 후 위험 요소 출현
3. 탭 시 → 반응 시간 기록, 성공 피드백
4. 3000ms 초과 시 → 실패, 자동 다음 라운드

**저장 데이터:** `hazardAnswers: boolean[]` (3개)

**완료 후:** `/result-loading` 이동

## 5. 공통 컴포넌트

### `TestProgressBar`
- Props: `current: number`, `total: number`, `label: string`
- 상단 고정 (`sticky top-0`), 고대비 노란색 채우기

### `SpeechGuide`
- Props: `text: string`, `autoPlay?: boolean`
- 마운트 시 자동 TTS (`autoPlay` 기본 `true`)
- 재생 버튼 UI (🔊 아이콘 + "다시 듣기")
