# Phase 3 기능 명세서 — 광고 수익화 페이지

> 작성자: 기획/설계 에이전트 | 2026-03-17

---

## 핵심 전략

테스트 완료 직후 사용자는 결과에 대한 궁금증이 최고조에 달한다.
이 심리를 활용하여 `/result-loading` 에서 3~5초 강제 대기 + 광고 노출로
자연스러운 광고 수익을 확보한다.

---

## 1. `/result-loading` 페이지 전체 설계

### 레이아웃 (위 → 아래)

```
┌─────────────────────────────────┐
│  [분석 중 텍스트 + 스피너]       │  ← 상단 1/3
│                                 │
│  ┌─────────────────────────┐    │
│  │   [AdBanner 광고 영역]   │    │  ← 중앙 (300×250 or 320×100)
│  └─────────────────────────┘    │
│                                 │
│  [카운트다운: "3초 후 결과 공개"] │  ← 하단 1/3
│  [진행 바 애니메이션]            │
└─────────────────────────────────┘
```

### 타이머 플로우

- 페이지 진입 즉시 5초 카운트다운 시작
- 광고 로드 여부와 무관하게 5초 후 자동으로 `/result` 이동
- TTS: "검사 결과를 분석하고 있습니다. 잠시만 기다려주세요."

### 애니메이션

- 스피너: 노란색 테두리 회전 (80px)
- 분석 문구: 3개 메시지 순환 (1.5초 간격)
  1. "어르신의 반응 속도를 분석하고 있습니다..."
  2. "표지판 인식 능력을 평가하고 있습니다..."
  3. "종합 결과를 정리하고 있습니다..."
- 카운트다운 바: 좌 → 우로 채워지는 노란색 바 (5초 동안)

## 2. AdBanner 컴포넌트 설계

### 기술 요구사항

| 항목 | 내용 |
|------|------|
| 광고 스크립트 | `layout.tsx`에 `<Script strategy="afterInteractive">` 로 삽입 |
| 광고 단위 | Google AdSense `<ins class="adsbygoogle">` |
| 사이즈 | 모바일 반응형 (300×250 기본, 뷰포트 좁으면 320×100) |
| Publisher ID | 환경변수 `NEXT_PUBLIC_ADSENSE_CLIENT` (빌드 시 주입) |
| 오류 처리 | `ErrorBoundary`로 감싸 광고 실패가 앱을 깨뜨리지 않도록 |
| Fallback | 광고 실패 시 `SafeDrivingCard` 렌더링 |
| 개발 환경 | `NODE_ENV === 'development'` 이면 광고 대신 플레이스홀더 표시 |

### 렌더링 흐름

```
<AdErrorBoundary fallback={<SafeDrivingCard />}>
  <AdBanner />           ← 실제 AdSense 삽입
</AdErrorBoundary>
```

## 3. SafeDrivingCard 컴포넌트 설계

광고 로드 실패 또는 개발 환경에서 표시할 대체 컨텐츠.
노출 시간 동안 사용자에게 유익한 정보를 제공하여 이탈 방지.

**5가지 안전 운전 수칙 (카드 형태):**

1. 🛑 **출발 전 확인** — 좌우 사이드미러, 안전벨트 착용 확인
2. 🚗 **속도 조절** — 고령 운전자는 반응 속도를 고려해 여유 있게 운전
3. 🌙 **야간 주의** — 해질 무렵과 야간에는 전조등을 켜고 속도를 줄이기
4. 🅿️ **주차 확인** — 주차 후 핸드브레이크 체결 및 기어 P 확인
5. 💊 **약 복용 주의** — 졸음 유발 약 복용 후 운전 금지

카드는 5초 동안 1개씩 순환 표시.
